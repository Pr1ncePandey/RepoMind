# Elon Musk 7-Step Product Framework

This framework is used to design and improve products efficiently.

Claude should follow these steps sequentially when analyzing or optimizing a project.

---

## Step 1 — Question Every Requirement

Ask why each feature exists.

- Who requested it?
- Is it truly necessary?
- Can the product function without it?

Goal: eliminate unnecessary complexity.

---

## Step 2 — Delete the Part or Process

Remove components that do not contribute directly to the core function.

If you are not adding things back later, you likely didn’t remove enough.

Goal: reduce system complexity.

---

## Step 3 — Simplify and Optimize

After removing unnecessary components, simplify what remains.

Focus on:

- simpler architecture
- fewer dependencies
- easier maintainability

Goal: make the system elegant.

---

## Step 4 — Accelerate Development

Increase iteration speed.

Focus on:

- faster build pipelines
- shorter feedback loops
- rapid prototyping

Goal: shorten development cycles.

---

## Step 5 — Automate

Only automate **after** simplification.

Automate repetitive tasks such as:

- testing
- deployments
- data pipelines

Goal: increase efficiency.

---

## Step 6 — Optimize User Experience

Improve usability.

Consider:

- onboarding
- interface simplicity
- user friction

Goal: maximize user adoption.

---

## Step 7 — Scale the System

Finally evaluate scalability.

Check:

- infrastructure scaling
- cloud readiness
- SaaS deployment potential

Goal: prepare the product for large-scale usage.

---

## Output Format

When applying this framework Claude should generate:

PRODUCT OPTIMIZATION REPORT

1. Requirements to Question
2. Components to Remove
3. Simplification Opportunities
4. Development Speed Improvements
5. Automation Opportunities
6. UX Improvements
7. Scalability Plan
